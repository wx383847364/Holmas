# Holmas 核心找猫新手引导方案

## Summary

- 第一版采用“互动遮罩”而不是纯图文弹窗：首次进入 `Main` 主界面时自动出现，逐步高亮玩家真实会用到的控件。
- 引导范围只覆盖核心找猫：任务栏、开始/继续找猫、Walk/Find 模式、棋盘翻格、体力、自动领奖。
- 暂无图片资源时，先用半透明遮罩、描边高亮、猫图 fallback/简单图形占位；后续图片资源进来后只替换插图位，不改流程逻辑。

## Key Changes

- 新增一个 `Overlay` 类型页面：`tutorial.core_find_cat.overlay`，放在 `App.HotUpdate/Holmas/UI/Screens/Tutorial`。
- 新增 `TutorialOverlayController / TutorialOverlayView / TutorialOverlayVm`：
  - `Controller` 只控制步骤、下一步、跳过、完成，不写玩法规则。
  - `View` 只负责遮罩、高亮框、文案、按钮、占位图。
  - `Vm` 固定承载当前步骤标题、正文、按钮文本、目标区域、是否允许点击穿透。
- 在 `MainView` 暴露一组只读引导目标引用，例如任务栏区域、开始按钮、Walk/Find 切换、棋盘容器、体力区域；不要让 Tutorial 反向遍历私有 UI 结构。
- 在 `MainPageController.OnOpen/OnResume` 后触发一次检查：如果当前教程未完成且主界面已渲染，就调用 `ShowOverlayAsync("tutorial.core_find_cat.overlay", payload)`。
- 新增 HotUpdate UI 层教程进度存储：
  - 使用现有 `IPersistence` 独立 key：`holmas.tutorial.core_find_cat.v1`。
  - 只保存 `completed=true` 和 `completedAtUtcMilliseconds`。
  - 不写入 `HolmasGameplayRuntime`，不影响任务、地图、奖励、体力等玩法状态。
- 在主界面增加一个可重看入口，建议复用或新增一个小的“？”帮助按钮；点击后无视完成标记，重新打开同一教程 Overlay。

## Tutorial Steps

1. **任务目标**
   - 高亮任务栏前 2-4 个可见任务槽。
   - 文案：`这里显示正在寻找的猫、进度和金币奖励。找到对应猫后会自动推进，完成后自动领奖。`

2. **开始找猫**
   - 高亮 `开始找猫/继续找猫` 按钮。
   - 文案：`点击这里进入棋盘。未完成的棋盘会保留，下次回来可以继续找。`

3. **选择模式**
   - 高亮 `WalkToggle / FindToggle`。
   - 文案：`行走模式适合试探：翻到普通格不耗体力，遇到猫耗 2 点。寻找模式更直接：每翻一个有效格耗 1 点。`

4. **翻开棋盘**
   - 如果已有棋盘，高亮棋盘区域；如果还没开局，提示先点开始并等待棋盘出现后继续。
   - 文案：`点击格子寻找隐藏猫。数字会提示附近线索；找到本局全部猫后，会自动进入下一局。`

5. **体力提示**
   - 高亮体力区域。
   - 文案：`翻格需要体力。体力会随时间恢复；体力不足时不能继续翻格。`

6. **完成提示**
   - 不高亮具体控件，展示收束说明。
   - 文案：`核心目标很简单：接任务，找猫，拿金币。金币之后可以用于宣传升级，让侦探社继续成长。`

## Test Plan

- 单元测试教程存储：
  - 首次无数据时返回未完成。
  - 标记完成后再次读取返回已完成。
  - 存储损坏或空字节时按未完成处理，不阻塞主流程。
- UI/流程测试：
  - 新号首次进入 `Main` 后自动打开教程 Overlay。
  - 点击“下一步”按固定 6 步推进，最后关闭并写入完成标记。
  - 点击“跳过”关闭并写入完成标记。
  - 已完成玩家再次进入 `Main` 不自动弹出。
  - 点击帮助入口可以强制重看教程。
- 边界检查：
  - `Tutorial` 代码只在 `App.HotUpdate` UI 层，不改 `App.AOT`。
  - 不把教程完成状态塞进玩法 runtime。
  - Overlay 不进入 Page 历史栈，不影响 `BackAsync()` 的页面返回语义。
- 人工验收：
  - 竖屏真机或 Play Mode 下确认遮罩不挡住必须点击的按钮。
  - 确认中文文案不溢出，按钮在小屏上仍可点。
  - 确认暂无图片资源时占位表现可接受，后续图片替换位清晰。

## Assumptions

- 第一版不做强制“必须点真实格子才能过下一步”的硬教程，只做轻互动遮罩；这样实现风险低，也不会卡住玩家。
- 第一版不讲宣传升级、离线收益、广告槽完整规则，只在最后一句带到成长目标。
- 后续图片资源进入后，替换每一步的插图/猫图占位，不改变步骤顺序和存储 key。
- `Battle` 页保持兼容，不作为第一版新手引导主路径；当前正式体验以 `Main` 内嵌找猫为准。
