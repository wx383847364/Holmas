# Holmas v1 正式建筑内容表方案

## Summary

本方案将原“建筑功能”统一改为“宣传功能”。

`v1` 内每一级不再表示建筑阶段，而是表示“在一个城市开展宣传”：

- 总阶段数为 `100` 级
- 每一级对应 `1` 个城市宣传点
- 每一级固定有 `4` 个宣传升级按钮
- 当前级的 `4` 个宣传功能全部升满后，才能进入下一级城市

四个宣传功能固定为：

- `传单宣传`
- `广播宣传`
- `网络宣传`
- `电视宣传`

每个宣传功能固定可升级 `5` 次：

- 每次升级都需要消耗金钱
- 每次升级宣传，玩家固定获得 `1` 点经验
- 同一城市下，不同宣传功能可配置不同升级消耗

## 完成情况

- 当前状态：进行中
- 进度说明：已完成城市宣传内容表结构与校验规则定义，待运行时、导表链和主链语义完全收口。

## 设计目标

- 用“城市宣传”替代原“建筑升级”表达
- 把阶段推进扩展到 `100` 个城市，形成长线成长
- 每一级都提供统一的 `4` 类宣传入口，便于理解和扩展
- 所有宣传功能统一为 `5` 次升级，便于导表和校验
- 当前城市四项宣传全部满级后，解锁下一城市
- 把玩家经验来源明确收口为“宣传升级 +1 经验”

## 核心规则

### 1. 等级含义

- `Stage 1` 到 `Stage 100` 分别表示 `100` 个不同城市宣传点
- 玩家当前所在级，表示当前正在运营的城市
- 只有当前城市的四个宣传功能全部升满，才能进入下一个城市

### 2. 宣传按钮

每个城市固定包含以下 `4` 个按钮：

- `leaflet` -> `传单宣传`
- `radio` -> `广播宣传`
- `online` -> `网络宣传`
- `tv` -> `电视宣传`

### 3. 升级规则

- 每个宣传功能固定升级 `5` 次
- 每次升级都需要消耗金币
- 每次升级宣传，玩家固定获得 `1` 点经验
- 当前城市内任一宣传功能未满 `5` 级时，不允许跳到下一城市

### 4. 推荐运行时语义

当前方案建议将原有“建筑升级”语义平移为“宣传升级”：

- 升级接口仍然是“对某个宣传功能执行升级”
- 升级成功后：
  - 扣除金币
  - 对应宣传功能等级 `+1`
  - 玩家经验 `+1`
  - 记录当前城市宣传进度
- 当当前城市四项宣传全部达到 `5/5` 后，推进到下一城市

## 城市阶段设计

`v1` 固定为 `100` 个城市宣传阶段，建议直接使用城市命名。

命名规则建议：

- `Stage 1` 到 `Stage 100` 按城市列表顺序展开
- `stageName` 直接使用城市名
- `notes` 可标注该城市所处层级，如“新手城市 / 成长城市 / 高阶城市 / 核心城市”

## 正式内容表设计

建议将内容表改为“城市 + 4 类宣传功能”的结构。

推荐字段如下：

```text
agencyStageId,stageName,promotionIds,promotionLevelCaps,promotionUpgradeCosts,notes
```

字段说明：

- `agencyStageId`：城市阶段 ID
- `stageName`：城市名称
- `promotionIds`：当前城市固定的宣传功能 ID 列表
- `promotionLevelCaps`：四项宣传功能的最大等级，固定为 `5;5;5;5`
- `promotionUpgradeCosts`：每项宣传功能 `5` 次升级消耗的金币
- `notes`：备注

经验规则说明：

- 不再配置 `promotionEffectValues`
- 每次任意宣传功能升级成功，玩家固定获得 `1` 点经验
- 单个城市总共有 `4 x 5 = 20` 次宣传升级机会
- 因此单个城市总共可提供 `20` 点经验
- `100` 个城市总共可提供 `2000` 点经验

## 数值解读

### 每一级的固定逻辑

每一级都采用同一套结构：

- `1` 个城市
- `4` 个宣传按钮
- 每个按钮固定升级 `5` 次
- 每次升级都单独消耗金币
- 每次升级固定给玩家 `1` 点经验

### 阶段推进规则

- 当前城市 `leaflet / radio / online / tv` 全部达到 `5/5`
- 才能从当前 `Stage` 进入下一 `Stage`

### 数值配置含义

以 `Stage 1` 的 `传单宣传` 为例：

- 最大等级为 `5`
- 五次升级金币消耗分别为：`20 / 35 / 55 / 80 / 110`
- 五次升级全部完成后，累计提供 `5` 点经验

以单个城市整体计算：

- `4` 个宣传功能
- 每个功能 `5` 次升级
- 合计 `20` 次升级
- 因此该城市总计提供 `20` 点经验

## 100级扩展建议

为了把表稳定扩展到 `100` 级，建议遵循以下规则：

- `stageName` 使用真实城市名
- `promotionLevelCaps` 全部固定为 `5;5;5;5`
- 后面的城市可以整体抬高四类宣传的金币消耗
- `电视宣传` 和 `网络宣传` 后期成本可高于 `传单宣传` 与 `广播宣传`
- 所有城市统一遵循“每升级一次 +1 经验”

如果需要做城市梯度，可分为：

- `Stage 1-20`：新手城市
- `Stage 21-50`：成长城市
- `Stage 51-80`：高阶城市
- `Stage 81-100`：核心城市

## 数据约束

导表时建议增加以下校验：

- `promotionIds` 长度固定为 `4`
- `promotionLevelCaps` 必须固定为 `5;5;5;5`
- `promotionUpgradeCosts` 中每一组长度必须固定为 `5`
- 所有升级消耗必须大于 `0`
- 当前阶段四项宣传都满级后，才允许推进 `agencyStageId`
- 表内总阶段数必须为 `100`
- `agencyStageId` 必须连续为 `1-100`
- `stageName` 不允许重复

经验校验建议：

- 每次任意宣传升级成功，经验固定 `+1`
- 单个城市经验上限固定为 `20`
- 全表总经验上限固定为 `2000`

## 运行时约束

本阶段只要求完成语义替换和数据收口：

- 原“建筑”概念统一替换为“宣传功能”
- 原“建筑升级”改为“宣传升级”
- 每一级不再表示建筑群，而表示一个城市宣传点
- 当前城市固定只有四个宣传按钮
- 每个宣传按钮固定 `5` 次升级
- 每次宣传升级成功，玩家固定获得 `1` 点经验
- 四项宣传全部满级后才解锁下一城市

本阶段默认不做：

- 不新增第二种宣传货币
- 不新增宣传失败概率
- 不新增宣传效果值字段
- 不新增渠道联动加成
- 不新增随机事件修正
- 不新增 UI 特效规则
