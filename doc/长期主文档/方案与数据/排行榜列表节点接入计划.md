# 排行榜列表节点接入计划

## Summary

- 排行榜列表数据改为渲染到 `LeadbroadPanel/LeaderInfo/LeaderList`。
- 列表 item 模板固定使用 `LeadbroadPanel/LeaderInfo/LeaderList/GameObject/PlayerInfo`。
- `MyInfo` 只渲染自己的数据，Top 3 使用 `LeaderInfo/No.1~No.3`。

## Key Changes

- 更新 `LeaderboardBindings`：
  - `LeaderList` 绑定现有 `ScrollRect`。
  - `LeaderListContent` 绑定 `LeaderList/GameObject`。
  - `ItemTemplate` 绑定 `LeaderList/GameObject/PlayerInfo`。
  - `MyInfo` 绑定 `LeaderInfo/MyInfo`。
  - `Top1/Top2/Top3` 绑定 `LeaderInfo/No.1`、`No.2`、`No.3`。
- 更新 `LeaderboardView`：
  - 不再创建 `LeaderboardRuntimeOverlay/ListRoot/SelfText` 来承载列表。
  - 渲染时：
    - `entries[0..2]` 分别写入 `No.1~No.3`。
    - `entries[3..]` 写入 `LeaderList/GameObject` 下的复用 item。
    - item 复用 `PlayerInfo` 模板实例；不足时克隆模板，过多时隐藏。
    - `MyInfo` 始终显示 `selfEntry`，即使玩家不在 Top 20。
  - 切榜时清空/隐藏旧行并把 `LeaderList` 滚动位置重置到顶部。
- 列表策略：
  - 首期不做无限循环列表。
  - Top 20 使用模板池复用足够；后续如果做 100+、分页或无限滚动，再抽 `UiVirtualScrollList`。

## Test Plan

- `LeadbroadPanel` binding 测试确认能解析：
  - `LeaderInfo/LeaderList`
  - `LeaderInfo/LeaderList/GameObject`
  - `LeaderInfo/LeaderList/GameObject/PlayerInfo`
  - `LeaderInfo/MyInfo`
  - `LeaderInfo/No.1~No.3`
- 渲染测试覆盖：
  - Top 3 写入前三名节点。
  - 第 4 名以后写入 `PlayerInfo` 模板池。
  - 少于 3 条时未用节点清空或隐藏。
  - `MyInfo` 显示本人数据。
  - 切榜后旧行不残留、滚动回顶部。

## Assumptions

- 不改 prefab 节点名。
- `LeaderList/GameObject/PlayerInfo` 是唯一列表 item 模板。
- 当前只显示 Top 20，不实现无限循环列表。
