## Holmas 任务栏规则调整计划：广告槽到期改“做完再锁” + 任务完成自动领奖

### Summary
把当前任务栏规则从“广告槽到期即清空重锁、任务手动领奖”改成以下新行为：

- 广告槽解锁后仍然立即补任务，解锁时长仍由当前玩家等级配置里的 `AdUnlockHours` 决定。
- 广告槽到期后，如果槽里还有任务，不再立刻清空；玩家可以继续完成该任务。
- 所有槽位的任务在达到完成条件时自动领奖，只增加金币，不再保留手动领奖步骤。
- 自动领奖后槽位立即清空；如果槽位当前仍是解锁状态，则立刻补新任务。
- 用户已确认：局内自动完成后允许“局内立刻补新任务”，且新补进任务从 `0` 进度开始，不回补当前局之前已翻出的猫。

### Key Changes
#### Runtime / 任务规则
- 调整 `HolmasTaskRuntimeInstance` / `HolmasTaskProgressService` 的完成链路：
  - 任务一旦达到 `TargetCount`，立即结算金币并清槽，不再等待手动 `ClaimTaskReward`。
  - 自动领奖后，如果槽位仍 `IsUnlocked=true`，立即补新任务。
  - 新补进任务初始进度固定为 `0`，不回灌当前局历史已找到猫。
- 调整广告槽到期逻辑：
  - `RefreshExpiredAdSlots` 不再对“槽内仍有任务”的广告槽执行 `ClearSlot + LockSlot`。
  - 广告槽应进入“已过期但允许收尾”的状态：任务可继续推进，直到任务完成并自动领奖后，再执行重新锁定。
  - 默认开放槽保持不受广告到期逻辑影响。
- 调整 `HolmasGameplayRuntime`：
  - 去掉“未结束棋盘暂不能领奖”的业务前提，改为由自动领奖链处理，不再依赖手动点击领取。
  - 保留金币只作为任务奖励产出，不恢复经验发放。
  - 保留 `AdUnlockHours` 取值来源不变，继续走当前 meta/player level 配置。

#### 状态模型 / 接口
- 扩展槽位状态表达“广告已到期但任务未结清”的场景。
  - 推荐在 `TaskSlotState` 上新增显式布尔态，例如“广告已过期待锁定”，不要继续只靠 `UnlockExpireAt` 推断。
  - 自动领奖完成后，如果该标记为真，则先清槽再锁槽；否则按已解锁槽位立即补任务。
- `ClaimTaskReward` 相关接口保留兼容壳，但语义改为：
  - 运行时主链不再依赖它推进正常任务生命周期。
  - UI 点击任务槽时改为“查看状态/说明”，不再作为主领取入口。
  - 如果保留 `CanClaimReward` 字段，需同步改成只服务兼容层或迁移期，不再驱动主 UI 文案。

#### UI / Presenter
- 更新 `MainPresenter` 和 `AgencyMainPresenter` 的任务文案：
  - 去掉“点击领取”“领取奖励”等主文案。
  - 已完成态不再停留在可领奖状态，而应在自动结算后直接切成“新任务”或“已锁定/待锁定”。
- 更新 `MainPageController` / `AgencyMainPageController`：
  - 移除任务槽点击触发领奖的主流程。
  - 点击槽位时只展示说明性状态即可，例如“进行中”“广告槽已到期，完成当前任务后将重新锁定”。
- Main 内嵌棋盘保留现有入口，不重新引回 Battle 作为正式入口。

### Test Plan
- 纯逻辑测试：
  - 广告槽解锁后立即补任务，解锁时长仍来自 `AdUnlockHours`。
  - 广告槽到期但任务未完成：槽位不清空、不立刻上锁，任务仍可继续推进。
  - 广告槽到期后任务完成：自动发金币、清槽并重新锁定。
  - 默认槽任务完成：自动发金币、清槽并立即补新任务。
  - 广告槽任务完成且未到期：自动发金币、清槽并立即补新任务。
  - 局内立刻补新任务时，新任务进度从 `0` 开始，不回补本局已翻出的同类猫。
- Runtime/UI 回归：
  - Main/AgencyMain 不再显示“点击领取”入口，任务完成后状态会自动刷新。
  - 局内翻猫导致任务完成时，金币立即增长，任务栏立即切到新任务。
  - 广告槽处于“到期待锁定”时，UI 能正确显示该状态且不崩溃。
- 兼容性回归：
  - 现有 `AdUnlockHours` 配置读取测试继续通过。
  - 存档恢复后，如存在“广告已到期但任务未结清”状态，能正确恢复并继续完成后再锁。
  - 需要替换/更新现有“过期即清空重锁”“未完成不能领奖”“Main 点击领奖”相关测试断言。

### Assumptions
- 已确认采用的新规则：
  - 广告槽到期后“做完再锁”。
  - 所有槽位都自动领奖。
  - 局内允许立刻补新任务。
  - 局内新补任务从 `0` 进度开始，不回补本局历史已找到猫。
- 允许任务栏在当前棋盘进行中与当前棋盘猫池暂时不一致；这是本次规则调整的预期副作用，不再按上一轮“强一致”方案约束。
- 本轮优先改规则与状态展示，不额外引入新的弹窗、提示动画或独立奖励结算页。
