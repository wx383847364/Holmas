# QA：新会话如何自动维护文档

新开会话后，如果你只想继承长期主文档规则、不默认启动真实 subagent，开场直接说这句：

```text
按长期主文档规则执行。
```

这句属于默认轻量口令：

- 继承长期主文档里的规则
- 不需要再把细节重复写进启动口令
- 不默认启动真实 subagent
- 不默认开启“Agent 6 退回后自动补位”的闭环

如果你希望这轮默认按真实 subagent 自动闭环执行，开场直接说这句短口令：

```text
按长期主文档规则执行，默认启动subagent。
```

这句属于增强闭环口令，固定等价于：

```text
按长期主文档规则执行。
这轮默认启动真实 subagent，并开启默认真实 subagent 自动闭环。
如果 Agent 6 退回 findings 且当前没有原实现真实 subagent，就自动补起同职责真实 subagent 继续修，修完后默认交回同一个 Agent 6 复审。
```

这句固定表示：

- 默认启动真实 subagent
- 允许主线程维护线程级真实 agent 注册表
- 允许主线程同时维护线程级 helper role，并优先复用同职责实例
- 允许 Agent 6 退回后自动补位同职责实现线
- 默认继续同一个 Agent 6 复审，不停在“已分析，等用户决定”

默认已包含：

- 文档维护流程
- Git 提交建议
- 会话建议
- Agent 6 审查链路
- 同一个 Agent 6 的复审规则
- Agent 6 不在线时继续修复与验证、后续再回补复审的规则
- 一旦本会话出现过自动压缩背景信息，后续大任务收尾默认倾向建议新开会话的规则
- 同一个会话内连续完成两个以上大任务时，即使主题没变也默认倾向新开会话的规则
- subagent 遇到普通技术阻塞时默认继续修，只有边界/协议/职责变化才上报主线程分流的规则
- 有 subagent 在处理中时，按先后顺序对外显示等待提示的规则
- Agent 6 返回 findings 后，逐条回显问题给用户的规则
- Agent 6 独立审查线完成后，无论通过与否，都必须同步主线程并附审查结论与阻塞项的规则
- 固定三段收尾模板：文档维护、Git 提交建议、会话建议
- 默认常驻的 helper role：文档 / 主线判断 helper、规则 / 流程审计 helper

如果新会话里的 Codex 没按长期主文档规则执行，再单独补具体约束；不要把这些细规则重新塞回默认启动口令。

注意：

- `按长期主文档规则执行。` 不等于“已经有真实 Agent 1~6 实例”
- `按长期主文档规则执行。` 也不等于“helper role 一定已经有现成实例”；主线程应先查线程级总注册表，再决定复用或新开
- 只有你明确写了“启动真实 subagent”、`默认启动subagent`，或“开启默认真实 subagent 自动闭环”，主线程才允许真的开 subagent
- 只有开启增强闭环口令时，主线程才允许在 Agent 6 退回后自动补起同职责真实 subagent
- helper role 的职责模板、总注册表和复用/关闭规则，统一看 [线程级辅助 subagent 角色](/Users/bruce/work/Holmas/doc/长期主文档/协作与执行/线程级辅助 subagent 角色.md)
