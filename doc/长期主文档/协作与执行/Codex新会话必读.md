# QA：新会话如何自动进入工作流

新开会话后，如果你想先低 token 搞清楚当前主线，再在确认任务后自动进入执行，开场直接说这句：

```text
按长期主文档规则执行。
```

这句属于默认入口口令，固定分两阶段。

## 阶段 A：低 token 启动

主线程先只做轻量理解，不直接扫项目代码。

默认只读：

- [项目总览](/Users/bruce/work/Holmas/doc/长期主文档/项目总览.md)
- [迭代记录索引](/Users/bruce/work/Holmas/doc/迭代记录/迭代记录索引.md)
- 最新迭代记录中的 `当前状态`、`关键结论`、`风险与阻塞`、`下一步`、`给下一轮的人`

默认不做：

- 不通读整份最新迭代长日志
- 不通读全量协作文档
- 不默认扫项目代码
- 不在你还没确认任务前直接开工

这一阶段主线程固定输出：

- 当前主线
- 当前阻塞
- `1 ~ 3` 个任务建议
- 推荐先做哪个

## 阶段 B：确认后进入执行

只有你确认“开始执行某个任务”后，主线程才进入执行调度阶段。

进入执行阶段后，主线程固定要做：

- 再读 `doc/长期主文档/协作与执行/` 下与当前任务相关的编组、角色、验收和收尾规则
- 必须参考 [长期 subagent 编组方案](/Users/bruce/work/Holmas/doc/长期主文档/协作与执行/长期 subagent 编组方案.md) 和 [线程级辅助 subagent 角色](/Users/bruce/work/Holmas/doc/长期主文档/协作与执行/线程级辅助 subagent 角色.md)
- 先查线程级总注册表，再决定复用或新开
- 在 `主线程直做 / 只启 helper / 启动一个或多个真实 subagent` 之间自动判断
- 某条实现链达到阶段里程碑后，自动进入 `Agent 6 审查 -> 原实现线修复 -> 同一审查链复审` 的闭环，直到通过后才算该链收口

如果你已经确认任务明确，不想先走建议阶段，可以直接说：

```text
按长期主文档规则执行，默认启动subagent。
```

这句属于增强执行口令，表示：

- 跳过默认 briefing，直接进入执行调度阶段
- 主线程优先按长期编组方案判断应否拉起真实 subagent
- 仍然沿用同一套阶段里程碑审查闭环和收尾规则

默认已包含：

- 文档维护流程
- Git 提交建议
- 会话建议
- Agent 6 作为阶段里程碑审查门的链路
- 同一审查链优先复用原审查实例、必要时由同职责 reviewer 接手同一 `review_chain_id` 的规则
- Agent 6 不在线或同步等待超时时继续修复与验证、把状态标成 `待回补复审`、后续再回补复审的规则
- 一旦本会话出现过自动压缩背景信息，后续大任务收尾默认倾向建议新开会话的规则
- 同一个会话内连续完成两个以上大任务时，即使主题没变也默认倾向新开会话的规则
- subagent 遇到普通技术阻塞时默认继续修，只有边界/协议/职责变化才上报主线程分流的规则
- 有 subagent 在处理中时，按先后顺序对外显示等待提示的规则
- Agent 6 返回 findings 后，逐条回显问题给用户的规则
- Agent 6 独立审查线完成后，无论通过与否，都必须同步主线程并附审查结论与阻塞项的规则
- 固定三段收尾模板：文档维护、Git 提交建议、会话建议
- 默认常驻的 helper role：文档 / 主线判断 helper、规则 / 流程审计 helper

如果新会话里的 Codex 没按长期主文档规则执行，再单独补具体约束；不要把这些细规则重新塞回默认入口口令。

注意：

- `按长期主文档规则执行。` 不等于“已经有真实 Agent 1~6 实例”
- `按长期主文档规则执行。` 也不等于“helper role 一定已经有现成实例”；主线程应先查线程级总注册表，再决定复用或新开
- `按长期主文档规则执行。` 默认先走低 token briefing，不等于“新会话一上来就直接扫代码”
- 主线程只有在你确认开始执行某个任务后，才进入自动调度与执行阶段
- 如果你显式补充“这轮不要开 subagent”或“先只给建议”，该限制优先于默认入口口令
- helper role 的职责模板、总注册表和复用/关闭规则，统一看 [线程级辅助 subagent 角色](/Users/bruce/work/Holmas/doc/长期主文档/协作与执行/线程级辅助 subagent 角色.md)

# QA：如何确认进入执行阶段

```text
按长期主文档规则执行。
开始执行：……
如果需要，按长期 subagent 编组方案和线程级辅助 subagent 角色自动判断并启动合适的 helper 或真实 subagent。
按阶段里程碑进入 Agent 6 审查闭环，直到通过后再收尾。
```
