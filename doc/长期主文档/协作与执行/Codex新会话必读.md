# QA：新会话如何自动维护文档

新开会话后，让 Codex 每次完成一个任务都顺手维护文档，开场直接说这句：

```text
按长期主文档规则执行。
```

这句默认继承长期主文档里的规则，不需要再把细节重复写进启动口令。

默认已包含：

- 文档维护流程
- Git 提交建议
- 会话建议
- Agent 6 审查链路
- 同一个 Agent 6 的复审规则
- Agent 6 不在线时继续修复与验证、后续再回补复审的规则
- 一旦本会话出现过自动压缩背景信息，后续大任务收尾默认倾向建议新开会话的规则
- 同一天连续完成两个以上大任务时，即使主题没变也默认倾向新开会话的规则
- subagent 遇到普通技术阻塞时默认继续修，只有边界/协议/职责变化才上报主线程分流的规则
- 有 subagent 在处理中时，按先后顺序对外显示等待提示的规则
- Agent 6 返回 findings 后，逐条回显问题给用户的规则
- Agent 6 独立审查线完成后，无论通过与否，都必须同步主线程并附审查结论与阻塞项的规则
- 固定三段收尾模板：文档维护、Git 提交建议、会话建议

如果新会话里的 Codex 没按长期主文档规则执行，再单独补具体约束；不要把这些细规则重新塞回默认启动口令。
