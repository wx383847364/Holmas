# 产品目标与边界

## 产品目标

这套系统的目标不是“让 agent 一次性生成一个页面”，而是建立一条长期稳定的 `UI 生成流水线`：

- 吃设计图、标注包和规则
- 产出结构化 `UiPrefabSpec`
- 再由确定性生成器落成 Unity UGUI prefab 草稿
- 最终交给项目 UI 实现方接业务逻辑

## 第一版范围

- 只支持 Unity UGUI
- 只支持 `设计图 + 标注包 + 结构化 spec`
- 只生成 prefab 草稿、节点层级、组件、资源槽位和绑定清单
- 只做人审后生成
- 只在 Holmas 内部孵化，不单开独立仓库

## 第一版不做

- 不直接从图片跳到正式 prefab
- 不自动覆盖正式业务资产
- 不自动生成完整 Presenter 业务逻辑
- 不自动接通 Holmas gameplay
- 不支持 UI Toolkit 或跨引擎输出
- 不在 Holmas 业务目录中堆生成器实现

## 成功标准

- 系统文档、代码、脚本、测试、样例都有独立专区
- `DesignPacket / UiPrefabSpec / PrefabBindingManifest / ProjectUiProfile` 四个契约冻结
- 生成器主逻辑不依赖 Holmas gameplay asmdef
- Holmas 能以 adapter 方式接入这套系统
- 后续抽离时可以按专区和目录整体迁移

## 关键非目标

- 不是一次性的页面搭建脚本
- 不是 Holmas 专属 UI 框架
- 不是自动写玩法逻辑的通用 Agent
- 不是只靠图片猜布局的黑盒工具
