# UI 自动生成系统总览

这份专区是 Holmas 内部孵化 `UI prefab 自动生成系统` 的长期主文档入口。  
从这一版开始，系统方案、接口、Agent 编组、输入输出规格、校验策略和 Holmas 接入约束，统一收敛到这个目录，不再散落到现有 `方案与数据`、`协作与执行` 或业务 UI 文档里。

## 当前定位

- 当前阶段：隔离化孵化 v2
- 当前目标：竖屏小游戏版本地自动分析 CLI、EditorWindow 桥接入口和 Holmas 竖屏回归闭环落地
- 试点项目：Holmas
- 未来方向：在 Holmas 内持续迭代，成熟后抽成单独小项目
- 下一阶段方向：高精度 UI 识别与重建双输出链路 MVP

## 当前冻结决策

- 第一版只服务 Unity UGUI
- 第一版输入固定为 `设计图 + 标注包 + 结构化 spec`
- 第一版输出固定为 `prefab 草稿 + binding manifest`
- 流程固定为 `DesignPacket -> UiPrefabSpec -> prefab 草稿 + PrefabBindingManifest -> validation -> review`
- 生成器系统区固定落在 `Assets/Tools/UiPrefabGenerator`
- 项目态任务数据固定落在 `Assets/UiPrefabGeneratorData`
- Unity 主入口固定为 `UiPrefabGenerator/Window/Portrait Generator`
- 默认项目模板固定为 `Assets/UiPrefabGeneratorData/Templates/ProjectDefaults/holmas_portrait_wechat_default.json`
- Holmas 竖屏试点 profile 固定为 `holmas_ugui_portrait`
- Holmas 仅作为试点接入方，不作为系统本体目录
- 文档权威正文固定落在本专区，旧 `方案与数据` 路径只保留跳转页
- 执行组织固定为 `6 个 subagent + 2 个 repo-local skill`

## 当前完成情况

- 已落地竖屏小游戏默认模板，参考分辨率为 `1080 x 1920`
- 已落地 `EditorWindow -> request.json -> analysis artifacts -> prefab + generation_result.json` 的主入口
- 已落地本地 `run_task_auto_analysis.sh` CLI、临时工程 batch 分析入口和最小自动分析服务
- 已落地 `Portrait Generator` 中的“自动分析并回写结果”按钮，成功后会自动刷新分析结果
- 已落地 Holmas 竖屏 profile，输出目录为 `Assets/Res/Perfabs/Generated/Holmas/Portrait`
- 已把模板、任务、缓存等项目态数据从模块代码目录剥离到 `Assets/UiPrefabGeneratorData`
- 已支持本地规则化自动产出 `DesignPacket / UiPrefabSpec / resource_match_report / analysis_result / analysis_summary`
- 手动回写 `design_packet.json / ui_prefab_spec.json / resource_match_report.json / analysis_result.json` 仍保留为兜底链路
- 已冻结高精度双输出链路的长期方向，并明确首轮采用 evidence MVP + 6 个 subagent 编组推进
- 已新增 Mermaid 正文版流程图页，明确总流程、当前轮次范围和高精度前半段 MVP 的 6 个 subagent 分工
