# UI 自动生成系统总览

这份专区是 Holmas 内部孵化 `UI prefab 自动生成系统` 的长期主文档入口。  
从这一版开始，系统方案、接口、Agent 编组、输入输出规格、校验策略和 Holmas 接入约束，统一收敛到这个目录，不再散落到现有 `方案与数据`、`协作与执行` 或业务 UI 文档里。

## 当前定位

- 当前阶段：隔离化孵化 v2
- 当前目标：冻结执行派工单、双 skill、子目录独占边界和 Holmas 试点接入约束
- 试点项目：Holmas
- 未来方向：在 Holmas 内持续迭代，成熟后抽成单独小项目

## 当前冻结决策

- 第一版只服务 Unity UGUI
- 第一版输入固定为 `设计图 + 标注包 + 结构化 spec`
- 第一版输出固定为 `prefab 草稿 + binding manifest`
- 流程固定为 `DesignPacket -> UiPrefabSpec -> prefab 草稿 + PrefabBindingManifest -> validation -> review`
- 生成器系统区固定落在 `Assets/Tools/UiPrefabGenerator`
- Holmas 仅作为试点接入方，不作为系统本体目录
- 文档权威正文固定落在本专区，旧 `方案与数据` 路径只保留跳转页
- 执行组织固定为 `6 个 subagent + 2 个 repo-local skill`

## 本目录职责

- 维护系统本体方案
- 维护长期稳定接口与规则
- 维护 Agent 编排与协作方式
- 维护 Holmas 接入边界
- 维护执行派工单和 skill 触发口径

## 不在这里写什么

- 业务 UI 的页面文案细节
- Holmas 玩法实现
- 临时讨论稿
- 单轮迭代过程记录
