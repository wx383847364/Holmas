# UI 自动生成系统隔离化孵化方案 v2 执行派工单

## Summary

- 本方案固定采用 `6 个 subagent + 双 skill` 的执行组织方式。
- 权威正文固定落在 `doc/长期主文档/UI自动生成系统` 专区；旧 `方案与数据` 路径只保留跳转页。
- 写入边界按“子目录独占”冻结，避免后续并行执行时互相踩目录。
- 审查、退回与复审历史只进入 `doc/迭代记录`，不混入长期主文档正文。

## Skill 方案

### `ui-prefab-governance`

职责：

- 约束专区落位、文档隔离、subagent 派工格式、asmdef 分层、迁移边界、Holmas 试点角色
- 用于规划、改写长期方案、拆派工单、审边界、审目录污染风险

典型触发语句：

- “把 UI 自动生成系统方案改成可执行 subagent 派工单”
- “收紧 UI 自动生成系统的目录边界和写入范围”
- “检查这套 UI 生成系统方案是否符合长期主文档隔离规则”

### `ui-prefab-pipeline`

职责：

- 约束 `DesignPacket -> UiPrefabSpec -> PrefabBindingManifest -> validation`
- 用于 spec 建模、生成流程、manifest 规则、回归与校验策略

典型触发语句：

- “补 DesignPacket 和 UiPrefabSpec 的契约”
- “规划 prefab 草稿生成和 manifest 输出”
- “给 UI 自动生成系统补 validator 和 deterministic 回归规则”

### Skill 组合规则

- `Subagent 1`：必带 `ui-prefab-governance`
- `Subagent 2`：必带 `ui-prefab-pipeline`
- `Subagent 3`：必带 `ui-prefab-pipeline`
- `Subagent 4`：必带 `ui-prefab-governance + ui-prefab-pipeline`
  - 只有触碰 Holmas 接入代码时才叠加 `unity-hotupdate-boundary`
- `Subagent 5`：必带 `ui-prefab-pipeline`
- `Subagent 6`：必带 `ui-prefab-governance`
  - 按审查对象叠加 `ui-prefab-pipeline`

## Subagent 切分

### Subagent 1 / Foundation-Contracts

目标

冻结专区正文位置、asmdef 分层、核心不可变契约和 adapter 接口边界。

允许写入范围

- `doc/长期主文档/UI自动生成系统`
- 旧稿跳转页
- `Assets/Tools/UiPrefabGenerator/Runtime/Core/Contracts`
- `Assets/Tools/UiPrefabGenerator/*.asmdef`

禁止写入范围

- `Runtime/Core/Intake`
- `Runtime/Core/Manifest`
- `Runtime/HolmasAdapter`
- `Editor/Generation`
- `Editor/Validation`
- `Tests`

交付物

- `10_隔离化孵化方案_v2_执行派工单.md`
- 四个核心契约矩阵
- asmdef 依赖方向说明
- `contracts freeze note`

验收点

- 只定义接口边界和不可变契约
- 不实现 Holmas profile 细节
- 权威正文只保留专区一份

### Subagent 2 / Design-Intake-Spec

目标

冻结 `DesignPacket -> UiPrefabSpec` 的字段、未决项表达、审阅稿格式和输入校验。

允许写入范围

- `doc/长期主文档/UI自动生成系统/04_输入规格_DesignPacket.md`
- `doc/长期主文档/UI自动生成系统/05_中间规格_UiPrefabSpec.md`
- `Runtime/Core/Intake`
- `Documentation~` 中与 intake/spec 相关的说明
- `Samples~/Holmas` 中的输入样例

禁止写入范围

- `Runtime/Core/Contracts`
- `Runtime/Core/Manifest`
- `Editor/*`
- `Tests`
- `HolmasAdapter`

交付物

- `DesignPacket` 字段表
- `UiPrefabSpec` 结构规范
- 最小样例 spec 审阅稿

验收点

- generator 和 validator 只消费 `UiPrefabSpec`
- 缺字段、命名冲突、状态不闭合可前置报错

### Subagent 3 / Generator-Manifest

目标

冻结 prefab 草稿生成步骤、节点命名、组件白名单、资源槽位规则和 `PrefabBindingManifest`。

允许写入范围

- `doc/长期主文档/UI自动生成系统/06_输出规格_PrefabBindingManifest.md`
- `doc/长期主文档/UI自动生成系统/07_生成流程与审阅流程.md`
- `Runtime/Core/Manifest`
- `Editor/Generation`
- `Documentation~` 中与生成相关的说明
- `Samples~/Holmas` 中的输出样例

禁止写入范围

- `Runtime/Core/Contracts`
- `Runtime/Core/Intake`
- `Editor/Validation`
- `Tests`
- `HolmasAdapter`
- Holmas 业务目录

交付物

- manifest 字段规范
- 生成步骤和白名单规则
- `sample manifest`

验收点

- 只输出 prefab 草稿 + manifest
- 不写业务 Presenter
- 同一 spec 重复生成稳定

### Subagent 4 / Holmas-Adapter-Profile

目标

实现 Holmas 侧 `ProjectUiProfile`、目录映射和 manifest 消费约束，不回流污染核心系统。

允许写入范围

- `doc/长期主文档/UI自动生成系统/09_Holmas接入约束.md`
- `Runtime/HolmasAdapter`
- `Documentation~` 中与 Holmas 接入相关的说明
- `Samples~/Holmas` 中的 Holmas 接入样例

禁止写入范围

- `Runtime/Core/*`
- `Editor/*`
- `Tests`
- `Assets/HotUpdateContent/Script/App.HotUpdate/Holmas/UI`
- `Assets/Scripts/App.Shared`

交付物

- `ProjectUiProfile` 职责与字段
- Holmas 输出目录约定
- manifest 消费边界说明

验收点

- 只做 Holmas 侧 profile、目录映射和消费约束
- 不定义核心 schema
- 不承载 gameplay 业务逻辑

### Subagent 5 / Validation-Regression

目标

冻结 spec 校验、输出校验、golden case 和 deterministic 回归规则。

允许写入范围

- `doc/长期主文档/UI自动生成系统/08_校验与回归策略.md`
- `Editor/Validation`
- `Tests`
- `Documentation~` 中与验证相关的说明
- `Samples~/Holmas` 中的 golden fixtures

禁止写入范围

- `Runtime/Core/*`
- `Runtime/HolmasAdapter`
- Holmas 业务目录

交付物

- 校验清单
- 回归规则
- `validation baseline`

验收点

- 非法节点、非法组件、缺资源位、命名冲突可前置拦截
- 至少有 1 份 approved sample spec 和 1 份 sample manifest 后再全面介入

### Subagent 6 / Review-Acceptance

目标

独立审查派工单和双 skill 方案是否可执行，给出通过 / 退回裁定。

允许写入范围

- `doc/迭代记录`
- review note / 审查附录

禁止写入范围

- 长期主文档正文
- `10_隔离化孵化方案_v2_执行派工单.md`
- 任何生产实现目录

交付物

- 审查结论
- findings 列表
- 复审条件

验收点

- 审查历史只落迭代记录
- 不把阶段性 findings 混进长期规则正文

## 启动顺序

1. `Subagent 1` 先冻结契约和目录边界，交付 `contracts freeze note`
2. `Subagent 2 / 3 / 4` 并行推进
3. `Subagent 3` 至少交付 1 份 `sample manifest`
4. `Subagent 5` 在至少有 1 份 approved sample spec 和 1 份 sample manifest 后全面介入，并产出 `validation baseline`
5. `Subagent 6` 只在阶段里程碑后介入审查
