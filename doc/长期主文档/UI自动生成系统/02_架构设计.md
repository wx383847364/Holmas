# 架构设计

## 分层结构

### 1. Editor Entry Layer

负责 Unity 内的用户入口。

- `EditorWindow`
- 模板选择与保存
- 请求创建
- 分析结果读取
- 生成结果展示

这一层不直接理解图片语义，也不直接写业务逻辑，只负责把任务组织成稳定的文件协议。

### 2. Design Intake

负责接收设计输入：

- 设计图
- 状态图
- 交互说明
- 资源命名
- 标注包

输出统一的 `DesignPacket`。

### 3. Spec Layer

负责把设计输入解释成稳定、可版本化、可 diff 的 `UiPrefabSpec`。

这层是机器权威中间层，后续生成和校验都依赖它，不直接依赖图片。

### 4. Generation Layer

负责把已批准 spec 转成 Unity UGUI prefab 草稿和 `PrefabBindingManifest`。

第一版优先保证：

- 目录隔离
- 节点命名稳定
- 组件白名单可控
- 重复生成可回归

### 5. Validation Layer

负责校验：

- spec 是否完整
- prefab 草稿是否符合规则
- binding manifest 是否齐全
- 同一 spec 的重复生成是否稳定

### 6. Project Data Layer

负责存放模板、请求、分析结果、生成结果等项目态数据。

- 这层固定放在 `Assets/UiPrefabGeneratorData`
- 不属于模块代码目录
- 未来模块拆出时可保留在项目内，不阻断工具整体迁移

```text

```

## 隔离原则

- 系统代码不写进 `Assets/HotUpdateContent/Script/App.HotUpdate/Holmas/UI`
- 系统代码不写进 `Assets/Scripts/App.Shared`
- Holmas 适配层单独放在 `HolmasAdapter`
- 项目态任务数据单独放在 `Assets/UiPrefabGeneratorData`
- 文档正文只写在当前专区

## asmdef 分层

- `UiPrefabGenerator.Core`
- `UiPrefabGenerator.Editor`
- `UiPrefabGenerator.Tests`
- `UiPrefabGenerator.HolmasAdapter`

依赖方向固定为：

```text
HolmasAdapter -> Core
Editor -> Core
Tests -> Core + Editor + HolmasAdapter
```

不允许：

- `Core -> HolmasAdapter`
- `Core -> App.HotUpdate`
- `Core -> App.Shared`
- `Editor -> App.HotUpdate`

## 竖屏默认落地

- 默认模板：`Assets/UiPrefabGeneratorData/Templates/ProjectDefaults/holmas_portrait_wechat_default.json`
- 默认方向：`portrait`
- 默认参考分辨率：`1080 x 1920`
- 默认输出目录：`Assets/Res/Perfabs/Generated/Holmas/Portrait`
- 默认 Unity 入口菜单：`UiPrefabGenerator/Window/Portrait Generator`
