# 架构设计

## 四层结构

### 1. Design Intake

负责接收设计输入：

- 设计图
- 状态图
- 交互说明
- 资源命名
- 标注包

输出统一的 `DesignPacket`。

### 2. Spec Layer

负责把设计输入解释成稳定、可版本化、可 diff 的 `UiPrefabSpec`。

这层是机器权威中间层，后续生成和校验都依赖它，不直接依赖图片。

### 3. Generation Layer

负责把已批准 spec 转成 Unity UGUI prefab 草稿和 `PrefabBindingManifest`。

第一版优先保证：

- 目录隔离
- 节点命名稳定
- 组件白名单可控
- 重复生成可回归

### 4. Validation Layer

负责校验：

- spec 是否完整
- prefab 草稿是否符合规则
- binding manifest 是否齐全
- 同一 spec 的重复生成是否稳定

## 目录结构

代码区固定为：

```text
Assets/Tools/UiPrefabGenerator
├── Runtime
│   ├── Core
│   └── HolmasAdapter
├── Editor
├── Tests
├── Samples~/Holmas
└── Documentation~
```

文档区固定为：

```text
doc/长期主文档/UI自动生成系统
```

## 隔离原则

- 系统代码不写进 `Assets/HotUpdateContent/Script/App.HotUpdate/Holmas/UI`
- 系统代码不写进 `Assets/Scripts/App.Shared`
- Holmas 适配层单独放在 `HolmasAdapter`
- 文档正文只写在当前专区

## asmdef 分层

- `UiPrefabGenerator.Core`
- `UiPrefabGenerator.Editor`
- `UiPrefabGenerator.Tests`
- `UiPrefabGenerator.HolmasAdapter`

依赖方向固定为：

```text
HolmasAdapter -> Core
Editor -> Core
Tests -> Core + Editor + HolmasAdapter
```

不允许：

- `Core -> HolmasAdapter`
- `Core -> App.HotUpdate`
- `Core -> App.Shared`
