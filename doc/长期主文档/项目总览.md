# 项目总览

这份文档是长期主文档体系中的第一入口。  
它不负责展开所有细节，而是帮助任何接手项目的人快速理解：项目是什么、现在做到哪了、先看什么、接下来该从哪里开始。

## 项目定位

- 项目名称：Holmas
- 中文名：福尔猫斯-寻猫侦探社
- 核心玩法：基于异形扫雷棋盘的“找猫 + 推任务 + 侦探社成长”
- 技术路线：Unity / 团结引擎 1.8、HybridCLR、YooAssets、HotUpdate 正式业务分层

- 当前阶段：进行中

- 当前建议起点：Agent 3：任务与长期进度

- 已明确默认先按 5-Agent 实操版协作，再视情况扩到 8-Agent 长期版

## 稳定边界

- `App.AOT`：宿主、资源、持久化、平台、HybridCLR
- `App.Shared`：稳定 DTO / 接口 / 事件
- `App.HotUpdate`：正式玩法和业务编排
- `MinesweeperTerrainData`：地图模板输入
- YooAssets：正式运行时资源入口
- UGUI：表现层，不承载核心玩法规则

## 当前实现主线

1. 由 CSV 导出地图、猫、任务、玩家等级、成长和建筑配置到 json / bytes
2. 在 Bootstrap 中从导出配置恢复 Catalog，并组装任务、地图、成长与建筑服务
3. 任务与地图继续消费 `playerLevel`，地图生成仍优先使用当前任务栏猫种池
4. 建筑升级统一消耗金币，每次成功升级固定提供 1 点玩家经验并重算 `playerLevel`
5. 任务领奖与离线结算只增加金币；地图完成只推进任务、统计与局内结算
6. 在无 UI 前提下由组合层统一暴露补任务、开地图、领奖、离线结算和建筑升级入口
7. 先由 Agent 5 做专项测试与回归验证，再交给 Agent 6 做挑刺审查
