# 项目总览

这份文档是长期主文档体系中的第一入口。  
它不负责展开所有细节，而是帮助任何接手项目的人快速理解：项目是什么、现在做到哪了、先看什么、接下来该从哪里开始。

## 项目定位

- 项目名称：Holmas
- 中文名：福尔猫斯-寻猫侦探社
- 核心玩法：基于异形扫雷棋盘的“找猫 + 推任务 + 侦探社成长”
- 技术路线：Unity / 团结引擎 1.8、HybridCLR、YooAssets、HotUpdate 正式业务分层

## 当前阶段

- 当前阶段：进行中
- 当前主线：后续进入实现时，先新增 Holmas_MetaLevelTable.csv，再把成长表接入 CSV转二进制 和 MetaProgression 配置消费链。；后续如果某轮确实启动了特定 Agent，在收尾时显式传 --agent-status 覆写对应行。
- 当前建议起点：Agent 3：任务与长期进度

## 当前范围

- 已明确地图表、猫表、任务表、玩家等级表
- 已明确任务栏 5 槽规则、广告解锁规则、普通任务奖励公式
- 已明确地图模板来自 `MinesweeperTerrainData`
- 已明确 `HolmasGameplayRuntime` 负责在无 UI 前提下串联地图完成、任务推进与长期进度
- 已明确 skill、subagent、文档维护三套协作机制
- 已明确默认先按 5-Agent 实操版协作，再视情况扩到 8-Agent 长期版

## 稳定边界

- `App.AOT`：宿主、资源、持久化、平台、HybridCLR
- `App.Shared`：稳定 DTO / 接口 / 事件
- `App.HotUpdate`：正式玩法和业务编排
- `MinesweeperTerrainData`：地图模板输入
- YooAssets：正式运行时资源入口
- UGUI：表现层，不承载核心玩法规则

## 当前实现主线

1. 冻结 Shared DTO 与 HotUpdate 入口骨架
2. 接入 `MinesweeperTerrainData -> BoardTemplate -> LevelSnapshot`
3. 实现任务栏、奖励公式、任务推进
4. 接入侦探社成长、广告槽位、离线收益
5. 在无 UI 前提下接入地图完成 -> 任务推进 -> 长期进度组合层
6. 由 Agent 5 做单元测试、校验和回归验证
7. 在组合层和验证稳定后再启动 UI 联调
