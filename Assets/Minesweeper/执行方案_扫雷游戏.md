# 扫雷游戏（类 Windows 版）执行方案

## 一、项目概述

- **目标**：在 Unity 2D 中实现类似 Windows 自带扫雷的小游戏。
- **平台**：Unity（UI + C#），可发布 PC/移动端。
- **核心玩法**：格子网格 + 左键翻开 / 右键插旗，数字表示周围雷数，踩雷即失败，翻开所有非雷格即胜利。

---

## 二、核心规则（对齐 Windows 扫雷）

| 规则 | 说明 |
|------|------|
| 左键点击 | 翻开格子；若为 0（周围无雷）则自动翻开周围格子（递归/队列展开） |
| 右键点击 | 插旗 / 取消插旗（标记疑似雷） |
| 首次点击 | 保证第一次点击的格子一定不是雷（可延迟生成雷区，或重新生成直到该格无雷） |
| 踩雷 | 翻开到雷 → 游戏结束，可显示全部雷 |
| 胜利 | 所有非雷格被翻开 → 胜利 |
| 难度 | 初级 9×9/10 雷、中级 16×16/40 雷、高级 30×16/99 雷（可配置） |

---

## 三、技术选型与资源

- **渲染**：Unity UGUI（Canvas + Grid Layout + 预制体格子）。
- **逻辑**：纯 C#，无物理引擎依赖。
- **资源**：
  - 可选：格子贴图（未翻开、翻开、数字 1–8、雷、旗、错雷等）。
  - 或纯 UI：用 Image/Button + Text/Sprite 拼出效果，用颜色区分状态。

---

## 四、模块划分与实现顺序

### 阶段 1：数据结构与雷区生成

1. **格子数据**
   - 每个格子：是否雷、周围雷数、是否已翻开、是否插旗。
   - 用二维数组或一维数组（行优先）存储，索引 `(row, col)` ↔ `index`。

2. **雷区生成**
   - 根据难度得到 `行数 R`、`列数 C`、`雷数 M`。
   - 在 `R×C` 中随机选 `M` 个位置放雷（用 HashSet 或 Fisher-Yates 洗牌，避免重复）。
   - **首次点击安全**：在玩家第一次点击后再生成雷，并确保该格不是雷；若该格被随机到则重新随机该格或重生成整盘。

3. **数字计算**
   - 对每个非雷格，统计 8 邻域内雷的个数，存为「周围雷数」。

### 阶段 2：核心逻辑

4. **翻开逻辑**
   - 若已翻开或已插旗：不处理（或可选：已插旗时不翻开）。
   - 若是雷：触发游戏结束。
   - 若是数字：仅翻开该格。
   - 若是 0：递归或 BFS 翻开所有相邻的 0 及与 0 相邻的数字格（经典「一片空白」展开）。

5. **插旗逻辑**
   - 仅对未翻开格子：切换「插旗」状态，更新剩余雷数显示（可选）。

6. **胜负判定**
   - 翻开到雷 → 失败。
   - 每当翻开一个格子后，检查：已翻开数量 == 总格子数 - 雷数 → 胜利。

### 阶段 3：UI 与交互

7. **场景与 Canvas**
   - 一个游戏场景，一个 Canvas（Screen Space Overlay 或 Camera）。
   - 顶部：难度下拉/按钮、剩余雷数、计时器（可选）、重新开始按钮。
   - 中间：雷区（Grid Layout Group 或代码排布格子）。

8. **格子预制体**
   - 一个格子预制体：背景 Image + 文字/数字（或 Sprite）+ 旗的 Image（默认隐藏）。
   - 状态：未翻开 / 已翻开（显示数字或空）/ 雷 / 插旗。
   - 用 Button 或 EventTrigger：左键 Click、右键 ContextMenu 或自定义右键检测。

9. **雷区生成（表现层）**
   - 根据 R、C 动态生成格子实例，挂载到 Grid 下；每个格子绑定 `(row, col)`，点击时回调到 GameManager。

10. **刷新显示**
    - 根据「是否翻开、是否雷、是否插旗、周围雷数」更新每个格子的显示（文本、图片、颜色）。

### 阶段 4：体验与收尾

11. **首次点击不踩雷**
    - 第一次点击时再生成雷区（且排除当前格），并重算所有数字，再执行翻开逻辑。

12. **计时器（可选）**
    - 从第一次点击开始计时，胜利/失败时停止。

13. **重新开始**
    - 清空/回收格子，重新生成雷区与 UI，重置状态。

14. **失败时展示**
    - 显示全部雷的位置，可把误插旗的格子标成「错旗」（如红色叉）。

## 五、推荐目录与脚本结构

```
Assets/
  Minesweeper/
    Scenes/
      Minesweeper.unity          # 游戏场景
    Prefabs/
      Cell.prefab                # 格子预制体
    Scripts/
      MinesweeperGame.cs         # 核心：雷区数据、生成、翻开、插旗、胜负
      CellView.cs                # 单个格子视图与输入
      GameUI.cs                  # 顶部 UI：难度、雷数、计时、重新开始
    Sprites/ (可选)
      cell_covered.png
      cell_revealed.png
      mine.png
      flag.png
      number_1~8 或 用 Text
```

- **MinesweeperGame**：持有二维格子数据、难度参数、当前状态（游戏中/胜利/失败）；提供方法：Init(difficulty)、Reveal(r,c)、ToggleFlag(r,c)、首次点击时生成雷区等。
- **CellView**：持有对 MinesweeperGame 的引用（或通过事件/委托），在点击时调用 Reveal/ToggleFlag，并根据 Game 返回的状态刷新显示。
- **GameUI**：创建雷区格子（实例化 Cell 预制体）、绑定难度与重新开始，显示雷数/计时。

## 六、实现顺序小结（按天/按步）

1. 搭场景与 Canvas，做格子预制体（先占位图或颜色即可）。
2. 实现 MinesweeperGame：格子数据结构、雷区生成（含首次点击安全）、数字计算。
3. 实现翻开与插旗逻辑，以及胜负判定。
4. 把 CellView 与 MinesweeperGame 接好：点击 → 调用逻辑 → 刷新格子显示。
5. GameUI：动态生成网格、难度选择、重新开始、雷数/计时。
6. 美术：替换为正式贴图或调色，失败时展示全部雷与错旗。
7. 简单测试：各难度、首次点击、插旗数量、胜利/失败分支。

## 七、可选扩展

- 音效：翻开、插旗、踩雷、胜利。
- 本地记录：最佳时间（按难度）。
- 自定义难度：输入行、列、雷数。
